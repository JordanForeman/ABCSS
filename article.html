What is HTML? by Jordan Foreman

What is HTML exactly? If you’ve been interested in learning website design for any amount of time, you’ve probably heard the word thrown around quite a bit already. HTML stands for HyperText Markup Language, and is a computer language that defines the content of our web sites.

Content is anything that a user can see and interact with - this includes things like text, buttons, links, images, etc. We use HTML to establish a hierarchy of these content elements. We’ll discuss the importance of a semantic HTML hierarchy in later chapters - for now, just remember that HTML defines the what of our web pages.

Getting Started

Hopefully at this point you’re asking yourself, “But how do I use HTML?”. The answer may surprise you, as actually using HTML is incredibly easy! All you’ll need now is a generic text editor. If you’re on a Windows computer, Notepad works fine. OS X users will want to use TextEdit. Open one of those programs now.

Note: While both of these programs are great for getting started, there are literally hundreds of text editing applications that are intended for writing website code. An industry favorite is Sublime Text, which is available on both Windows and Mac. I cannot recommend it enough.

With your text editor of choice open, go ahead and type the following (you can copy+paste, but manually typing out everything will make the concepts stick so much quicker):

<!DOCTYPE html>
<html lang="en">
  <head>
    <title>My First Website</title>
  </head>

  <body>
  </body>
</html>

That’s not a whole lot of code right? Lets walk through it and discuss what it does:

<!DOCTYPE html>

This line is used by the browser (Internet Explorer, Safari, Chrome, etc.) to determine what rendering mode to display your page in. We won’t go into detail about the different rendering modes quite yet, just remember that moving forward all of your pages should have <!DOCTYPE html> as the very first line.

<html lang=”en”>
  ...
</html>

This is the HTML root element of your web page. All elements in your HTML document will be written in a similar fashion. We begin by opening our root element with <html lang=”en”>, and closing it with </html>. Lets shift gears and discuss the structure of these elements.

Element Structure

Everything in an HTML document is called an element. Elements correlate to the content that we discussed earlier in this chapter - links, images, text, buttons, etc. Elements have the following structure:

<a href="home.html">Home</a>

That was an anchor element. An anchor is just a fancy term for something that will direct the browser to a different location when they are clicked on.

Notice how the anchor is written a lot like the <html> element above. Again, all elements are written the same way: They have an opening - <a> - and a closing - </a> tag that tells us what type of element we’re working with, and then some contents in the middle.

In the case of an anchor element, the text content in the middle is what will be what is displayed as a link. This is a link.

But what about that href=”home.html” part? This is what is known as an element property. Think of a property as anything that is special about any specific element. We set properties by saying property=”value” inside of the element’s opening tag. Different types of elements have different properties for you to customize. In the case of <a> elements, href is the most important - it determines where the user will be navigated to when they click on that link. In our example above, when a user clicks the linked text “Home”, their browser will be told to go load a “home.html” page.

Elements can also contain other elements: Suppose you had a paragraph of text with several links and an image inside of it? Here’s how that would look:

<p>
  Here is a bunch of text that I just typed up! 
  Click here for the home page: <a href="home.html">home page</a>!
  <img src="images/my_image.jpg">
</p>

You’ll notice a couple of new elements in the example above. <p> and <img>. A <p> element is a paragraph. Paragraphs are blocks of text content, similar to what you’re reading now - but you probably already knew that.

<img> elements are images that will be displayed on your page. This could be a snapshot of your dog or even a photo from that trip to France you took last year. Images also have a src property. This property tells the page where to look to find the image file to display. We’ll go into more depth on that and the href property in another chaper.

You’ll also notice that we don’t ever close an <img> element. Because images won’t ever contain another HTML element or text, they are allowed to close themselves. There are a handful of other elements that also don’t require closing tags. These are called self-closing elements. We’ll get to those later.

Putting it all Together

Lets take a step back and look at our HTML document again:

<!DOCTYPE html>
<html lang="en">
  <head>
    <title>My First Website</title>
  </head>

  <body>
  </body>
</html>

Hopefully now this makes a bit more sense; First, we have an HTML element with a lang attribute set to en. This tells the browser that our HTML document is an english language document.

Inside of our HTML, we have both a <head> and a <body> element, and our <head> contains a <title>, and that <title> contains the text “My First Website”.

That wasn’t so bad was it? If you’re still having a hard time following, don’t worry! HTML is ugly to read, and getting comfortable with all of the vocabulary will come with time.

In the next chapter, we’ll save this file and begin working on creating our first website! Cheers!
